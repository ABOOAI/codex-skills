# Content Rewrite Rules

Use this reference when converting source-image text into clearer HTML copy.

## Source Fact Inventory

Extract and preserve:

- Names: products, companies, plans, categories, people, places, source labels.
- Numbers: prices, limits, percentages, ages, dates, waiting periods, counts, rankings.
- Conditions: eligibility, renewal rules, exclusions, assumptions, coverage scope.
- Comparisons: better/worse, same/different, included/excluded, guaranteed/non-guaranteed.
- Caveats: uncertain OCR, missing rows, source notes, footnotes, exceptions.

Remove or downplay unless requested:

- Watermarks, platform slogans, promotional phrases, decorative labels, repeated headers.
- Source branding that does not affect the user decision.
- Redundant rows where every compared item is identical.

## Rewrite Ladder

Rewrite in this order:

1. Core claim: one sentence that explains what the page helps decide.
2. Viewer promise: one sentence that says what the reader can compare quickly.
3. Section map: short labels for the major groups.
4. Entity cards: one short positioning line per product or object.
5. Detail fragments: short table-cell statements that preserve exact facts.

## Compression Rules

- Prefer short, direct Chinese clauses over long source-style phrases.
- Keep one idea per line in tables.
- Use "结论 + 条件 + 限制" for dense benefits or rules.
- Turn slash-separated source text into explicit lists only when it improves clarity.
- Move long explanations out of cells and into note cards or detail panels.
- Keep exact numbers intact; do not round unless the source already rounds.
- Mark uncertain values with "待确认" instead of guessing.

## Sensitive Domains

For insurance, medical, legal, financial, and compliance content:

- Preserve caveats and exclusions.
- Avoid unsupported "best", "guaranteed", "risk-free", "must buy", or medical-outcome claims.
- Phrase recommendations as decision support: "更适合关注..." not "一定更好".
- Keep source-driven conditions visible near any conclusion.

## Copy Patterns

Use these patterns when they fit:

- "核心区别：A 在 X 上更强；B 的限制是 Y。"
- "适合：关注 X、预算 Y、能接受 Z 的用户。"
- "注意：该项仅在 X 条件下成立。"
- "相同点：四款产品均支持 X，因此不作为主要差异。"
- "待确认：原图该字段识别不清，需用户复核。"

## Example Rewrites

Dense:

```text
10年保证续保：医保内医疗费/质子重离子；非保证续保：医保外医疗费/癌症外购药/重疾特需
```

Clear:

```text
医保内医疗费、质子重离子可 10 年保证续保；医保外、外购药和重疾特需不保证续保。
```

Dense:

```text
30天，意外伤害无等待期
```

Clear:

```text
等待期 30 天；意外伤害无等待期。
```

Dense:

```text
医保内：5000元；医保外：1万
```

Clear:

```text
医保内一般医疗免赔 5000 元；医保外免赔 1 万。
```
