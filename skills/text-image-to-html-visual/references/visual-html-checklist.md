# Visual HTML Checklist

Use this checklist before final handoff.

## Information

- All visible source entities and important numbers are represented.
- The page separates facts from interpretation.
- Rewritten copy preserves the meaning of the original.
- Uncertain OCR/transcription items are marked for user confirmation.
- Removed source-brand text, watermark text, or platform claims when requested.

## Structure

- The first viewport explains what the artifact is and why it matters.
- Dense comparisons start with a "how to choose" or "key takeaway" section.
- Each product/entity has a positioning card.
- Key differences are shown before the exhaustive table.
- Detailed table rows are grouped and labeled.

## Visual

- Main text is readable at intended screenshot size.
- Table text does not overlap or clip.
- Color has semantic meaning and is used consistently.
- Enough white space exists between sections.
- The page does not rely on low-contrast gray text for key information.

## Screenshot

- Export width is large enough for dense content.
- Device scale factor is at least 2 for final PNG.
- Bottom whitespace is cropped.
- Final PNG opens correctly and dimensions are reported.
